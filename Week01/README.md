## OSI 7계층

다른 시스템간의 원활한 통신을 위해 **네트워크에서 통신이 일어나는 과정을 7단계로 나눈 표준 네트워크 모델**이다.

7계층은 **응용, 표현, 세션, 전송, 네트워크, 데이터링크, 물리계층**으로 나뉜다.
<br><br><br><br>
## 7계층으로 정의했을 때 장점

- 통신이 일어나는 과정을 단계별로 파악 가능
- 통신 중 이상이 생길 경우, 통신 장애를 일으킨 단계에서만 문제를 찾고 해결할 수 있음

> ### 1층: 물리계층
>
- **전송에 필요한 두 장치 간의 실제 접속과 절단 등 기계적, 전기적, 기능적, 절차적 특성에 대한 규칙을 정의한다.**
- 물리적 전송 매체와 전송 신호 방식을 정의한다.
- 데이터를 전기적인 신호로 변환해서 주고받는 기능을 진행하는 공간이다.
- 데이터를 전달하기만 하며, 어떠한 에러가 있는지에는 관여하지 않는다.
- 데이터 단위: 0과 1의 비트열(On/Off 전기적 신호 상태)
- 관련 장비: 통신케이블, 허브, 리피터 등
<br><br><br>
> ### 2층: 데이터링크계층

- **두 개의 인접한 개방 시스템들 간에 신뢰성 있고 효율적인 정보 전송을 할 수 있도록 한다.**
- 데이터링크 계층의 주요 기능
    - 흐름 제어 기능: 송신-수신 측의 속도 차이를 해결한다.
    - 프레임 동기화 기능: 프레임의 시작과 끝을 구분한다.
    - 오류제어 기능: 오류의 검출과 회복 기능을 한다.
    - 순서 제어 기능: 프레임의 순서적 전송 기능을 한다.
- 물리계층으로 송/수신되는 정보를 확인하고 오류없는 통신을 위해 여러 역할을 수행.
- MAC주소를 통해 통신
- 데이터 단위: 프레임
- 관련 장비: 브릿지, 스위치, 랜카드
<br><br><br>
> ### 3층: 네트워크계층

- **개방 시스템들 간의 네트워크 연결을 관리하는 기능과 데이터의 교환 및 중계 기능을 한다.**
- 네트워크 연결을 설정,유지,해제하는 기능
- **경로설정**, 트래픽 제어 등의 기능
- 데이터를 목적지까지 가장 안전하고 빠르게 전달하는 기능
- **라우터**를 통해 경로를 선택하여 **IP**주소를 지정하고 **경로에 따라 패킷을 전달**해준다.
- 데이터 단위: 패킷
- 관련 장비: 라우터
<br><br><br>
> ### 4층: 전송계층

- **논리적 안정과 균일한 데이터 전송 서비스르를 제공함으로써 종단 시스템(End-to-End)간에 투명한 데이터 전송을 가능하게 한다.**
- OSI 7계층의 상위 3계층과 하위 3계층의 **인터페이스 역할**을 한다.
- 주소설정, 오류제어, 흐름제어를 수행한다.
- **TCP, UDP등의 표준**이 있다.
- 두 호스트 시스템으로부터 발생하는 데이터의 흐름을 제공한다.
- 데이터 단위: 세그먼트
- 관련 장비: 게이트웨이
<br><br><br>
> ### 5층: 세션계층

- **송수신 측 간의 관련성을 유지하고 대화 제어를 담당한다.**
- **연결을 원하는 두 이용자 사이의 세션 설정 및 유지를 가능하게 해 줌으로써 두 이용자 간의 대화를 관리하고, 파일 복구 등의 기능을 지원한다.**
- 송수신 측 간의 대화 동기를 위해 전송하는 정보의 일정한 부분에 체크점을 두어 정보의 수신 상태를 체크하며, 이때의 체크점을 **동기점**이라고 한다.
    
    *동기점은 오류가 있는 데이터의 회복을 위해 사용하며, 소동기점과 대동기점으로 나눌 수 있다.
    
- 데이터 단위: 메시지
<br><br><br>
> ### 6층: 표현계층

- **응용계층으로부터 받은 데이터를 세션계층에 보내기 전에 통신에 적당한 형태로 변환하고, 세션계층에서 받은 데이터는 응용계층에 맞게 변환하는 기능을 한다.**
- 즉, 서로 다른 데이터 표현 형태를 갖는 시스템간의 상호 접속을 위해 필요한 계층이다.
- 코드변환, 데이터 암호화, 데이터 압축, 정보 형식 변환, 문맥 관리 기능을 한다.
- 데이터 단위: 메시지
<br><br><br>
> ### 7층: 응용계층

- **응용계층은 사용자(응용 프로그램)가 OSI환경(네트워크)에 접근할 수 있도록 서비스를 제공한다.**
- 응용프로세스간의 정보교환, 전자사서함, 파일전송,가상터미널 등의 서비스를 제공.
- 데이터 단위: 메시지

# OSI 7계층에서 사용된 네트워크 관련 장비

- 리피터 (물리계층)
    - 전송되는 신호가 전송 선로의 특성 및 외부 충격 등의 요인으로 인해 원래의 형태와 다르게 왜곡되거나 약해질 경우 원래의 신호 형태로 재생하여 다시 전송하는 역할을 수행.
    - 근접한 네트워크 사이에 신호를 전송하는 역할로, 전송 거리의 연장 또는 배선의 자유도를 높이기 위한 용도로 사용.
- 허브 (물리계층)
    - 한 사무실이나 가까운 거리의 컴퓨터들을 연결하는 장치로, 각 회선을 통합적으로 관리하며, 신호 증폭 기능을 하는 리피터의 역할도 포함
- 브리지 (데이터링크계층)
    - LAN과 LAN을 연결하거나 LAN안에서의 컴퓨터 그룹을 연결하는 기능
    - 네트워크 상의 많은 단말기들에 의해 발생되는 트래픽 병목현상을 줄일 수 있다.
- 스위치 (데이터링크계층)
    - 브리지와 같이 LAN과 LAN을 연결하여 더 큰 LAN을 만드는 장치
- 라우터 (네트워크계층)
    - 브리지와 같이 LAN과 LAN의 연결 기능에 데이터 전송의 최적 경로를 선택할 수 있는 기능이 추가된 것
    - 네트워크계층까지의 프로토콜 구조가 다른 네트워크 간의 연결을 위해 프로토콜 변환 기능을 수행
- 게이트웨이 (전송계층)
    - 전 계층(1~7계층)의 프로토콜 구조가 다른 네트워크의 연결을 수행한다.
    - 세션계층, 표현계층, 응용계층 간을 연결하여 데이터 형식 변환, 주소 변환, 프로토콜 변환 등을 수행한다
